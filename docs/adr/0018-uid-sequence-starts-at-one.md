# ADR-0018：UID 序列从 1 开始

- 状态：Accepted
- 日期：2026-07-18

## 背景

早期 Beta 的 `users_uid_seq` 从 1000 开始。产品决定公开 UID 与常见的社区编号保持一致，首位用户应为 `UID 1`。同时，UID 已被定义为公开、不可变且不可复用的身份标识，不能为了调整起点而重编已运营站点的用户。

## 决策

- 所有全新安装的 `users_uid_seq` 首个分配值为 `1`，后续单调递增。
- 追加迁移 `20260718070000_uid_sequence_starts_at_one`，不修改已冻结的历史迁移。
- 迁移只在序列仍处于历史初始状态且 `users` 为空时，将下一值设为 `1`。
- 已有用户的 UID 永不自动改写；升级时序列校正到现有序列值与最大 UID 中的较大值之后，避免重复分配。
- UID 继续为正整数、唯一、不可变且不可复用；UUID 仍是内部主键。

## 后果

- 新部署的首位管理员显示为 `UID 1`。
- 已在旧 Beta 版本创建账号的实例会保留原 UID（例如 1000）；若站点尚未对外运行且所有者明确需要重新编号，应先备份并采用一次性的人工迁移或重建空安装，不能作为普通升级行为。
- Docker 首次安装冒烟测试必须验证首位管理员 UID 为 `1`。
