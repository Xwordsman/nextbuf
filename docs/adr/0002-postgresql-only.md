# ADR-0002：PostgreSQL 是唯一官方数据库

- 状态：Accepted
- 日期：2026-07-12

## 背景

社区系统需要事务、复杂约束、全文搜索、游标分页、JSON 数据、审计和后续大表迁移。若 V1 同时兼容 MySQL 和 PostgreSQL，Schema、迁移、查询和测试都必须采用两者交集或维护两套实现。

## 决策

NextBuf V1 只官方支持 PostgreSQL 18。使用 Prisma 管理常规模型和查询，部分索引、全文搜索及 Prisma 难以表达的能力允许采用经过测试的 PostgreSQL SQL 迁移和参数化查询。

## 备选方案

### 同时支持 MySQL

用户选择更多，但会限制部分索引、搜索和约束能力，并使每次迁移的测试矩阵翻倍。当前没有足够需求承担该成本。

### SQLite 作为简单部署方案

单文件安装方便，但队列 Worker、并发写入、全文搜索和生产升级行为与 PostgreSQL 差异过大，容易形成一个实际上的第二产品。

### 完全依赖外部搜索服务

搜索能力强，但会给首版安装增加第五个必需服务，不符合可自托管和低门槛目标。

## 后果

正面影响：

- 可以充分使用约束、事务、`pg_trgm` 和全文检索。
- 迁移和查询只需维护一套生产语义。
- Docker 与外部托管数据库的行为更容易保持一致。

负面影响：

- 已有 MySQL 环境的站长需要新增 PostgreSQL。
- 导入其他社区数据时需要转换。
- PostgreSQL 主版本升级要单独提供备份恢复或 `pg_upgrade` 指南。

## 迁移与回退

未来若增加其他数据库，必须先抽象并验证全部查询、约束、迁移、搜索和测试，不以“ORM 声称支持”作为兼容完成标准。
