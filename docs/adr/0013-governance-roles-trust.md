# ADR-0013：治理、角色、制裁与信任等级

- 状态：已接受
- 日期：2026-07-16
- 对应版本：`v0.10.0`

## 背景

社区已经具备真实身份、内容、互动和通知，但尚不能安全处理举报、制裁和渐进式信任。管理角色、社区信任、专业声誉和未来交易信用来源不同；把它们合成一个等级会让活跃度意外获得删帖或封禁权限，也会使制裁无法解释和撤销。

## 决策

1. 每次举报保存独立来源、理由、目标快照、举报者当时 TL 和权重；同一举报者对同一目标最多一条未结举报，每日最多 10 条。
2. 同一目标的未结举报聚合到一个 ModerationCase。Case 只组织证据，不直接代表目标有罪；结案分 resolved 与 dismissed。
3. 所有处置写入不可变 ModerationAction，并保存操作者、当时角色、原因、目标、前后状态、请求 ID 和有效期。通用 GovernanceAuditEvent 另行覆盖角色和信任管理动作。
4. 警告、节点禁言、全站禁言、暂停和封禁保存为独立 Sanction。节点/全站禁言阻止新内容；暂停/封禁同时改变账号状态。撤销保留原记录和撤销人，不删除历史。
5. 基础用户是隐式角色。节点版主只管理所属节点内容和节点禁言；全局版主管理全站内容、警告、禁言和临时暂停；永久封禁、角色变更、规则激活和 TL4 只允许管理员。
6. TL0-TL3 自动计算，TL4 只能由管理员人工确认。TL 不授予版主、管理员、敏感数据或封禁能力；角色授权也不直接提高 TL。
7. 默认规则 v1 使用可重建 PostgreSQL 指标：账号时长、已读主题、已发布主题/回复、收到点赞和 180 天内未撤销制裁。阈值为：TL1=`1/3/1/0/0`，TL2=`14/20/10/3/0`，TL3=`60/100/50/20/0`。
8. 晋级立即生效；自动降级先进入 14 天宽限期，宽限期复算仍不达标才降级。有效制裁立即限制能力，不通过篡改 TL 历史实现。
9. 每次计算保存规则版本、指标、解释和变化历史。规则先创建 draft，完成 preview 批次后才能激活；激活会创建 apply 批次。批处理以 UID 游标和 Outbox 分片，Redis 清空后可恢复。

阈值顺序为：账号天数、已读主题数、发帖数（主题+回复）、收到点赞数、近期有效违规上限。

举报以 UTC 自然日计数。权重仅用于案件队列优先级：TL0/TL1 为 1，TL2/TL3 为 2，TL4 为 3；它不自动隐藏内容，也不代表举报成立。案件状态为 `open`、`in_review`、`resolved`、`dismissed`，结案会同时清空案件和来源举报的活动键，但不删除证据。

节点禁言阻止匹配节点的新主题、发布草稿、回复和回复草稿；全站禁言阻止全站内容创建，并额外阻止尚未绑定节点的附件上传。阅读、点赞、收藏和关注继续可用。暂停/封禁同时写 `User.status=suspended`，撤销或到期且不存在其他有效账号制裁时恢复 active。Worker 每分钟复核到期账号制裁；每次授权仍直接查询当前有效 Sanction，所以新增制裁无需等待 Worker。

信任 Worker 每个 Outbox 分片最多处理 25 个 UID。preview 只记录预测分布，不修改用户状态；apply 保存指标、解释、规则版本、宽限期和必要历史。批次总数、已处理数、变化数、游标和 `from/to/transition` 分布以 PostgreSQL 为事实。

## 权限矩阵

| 能力 | 用户 | 节点版主 | 全局版主 | 管理员 |
| --- | --- | --- | --- | --- |
| 举报公开目标 | 是 | 是 | 是 | 是 |
| 查看案件 | 否 | 所属节点 | 全站 | 全站 |
| 隐藏/恢复/关闭内容 | 否 | 所属节点 | 全站 | 全站 |
| 警告/节点禁言 | 否 | 所属节点 | 全站 | 全站 |
| 全站禁言/临时暂停 | 否 | 否 | 是 | 是 |
| 永久封禁 | 否 | 否 | 否 | 是 |
| 授予/撤销角色、TL4、激活规则 | 否 | 否 | 否 | 是 |

角色层级还要求：节点版主不能处置同节点版主、全局版主或管理员；全局版主不能处置其他全局版主或管理员；管理员可以处置全部范围，但不能撤销实例最后一个管理员。规则和角色 API 即使被直接请求也必须重新执行这些服务端检查。

## 备选方案

### 直接把 TL 当作管理权限

拒绝。阅读和互动指标可以被操纵，也无法表达节点作用域、人工任命和最后管理员保护；TL3 自动成为版主会违反最小权限原则。

### 只在 User 上保存 ban/mute 字段

拒绝。单字段不能同时表达节点禁言、全站禁言、临时暂停、永久封禁、多个并行制裁、撤销原因和历史证据。User.status 只作为账号级兼容状态，Sanction 才是制裁事实。

### Redis 保存举报限流、制裁或信任计数

拒绝作为事实来源。Redis 清空不能取消封禁、允许重复未结举报或让批次失去进度；当前规模下 PostgreSQL 行锁、唯一约束和可重建聚合足够。Redis/BullMQ 只承载可恢复执行。

### 规则激活后一次事务重算所有用户

拒绝。大事务会长期持锁、扩大失败回滚和部署风险。preview + UID 游标分片可以先看到影响，并允许 Worker 重启后继续。

## 后果

- 正面：所有举报、处置、角色和等级来源均可解释；制裁即时生效；规则可以预估、回放和逐步应用；TL、角色、专业声誉和交易信用保持独立。
- 代价：治理写入需要更多表和事务；信任指标读取会产生多次 PostgreSQL 查询；暂停到期后的 User.status 兼容字段最多在维护周期内恢复。
- 当前限制：`v0.10.0` 只提供聚焦案件页面和账户信任说明，不提供完整用户搜索、批量治理、角色/规则可视化配置、申诉流程或通用审计导出；这些属于 `v0.11.0` 或后续专门版本。

## 迁移与验证

- 迁移创建默认 active 规则 v1，为所有既有用户回填 TL0 状态，并用数据库触发器覆盖 Better Auth 邮箱、OAuth 和未来受控用户创建入口。
- 回填指标为零和 `source=initial`，随后由首个日常或规则 apply 批次从真实事实重算；不会把迁移时间伪装成用户行为。
- 数据库 CHECK 验证精确举报目标、案件活动键、制裁范围/时间、TL0-TL4、人工 TL4、规则/批次状态和计数关系；部分唯一索引限制单一 active 规则及同规则在途批次。
- 单元测试覆盖阈值、立即升级、人工 TL4 和降级宽限；真实 PostgreSQL 集成测试覆盖举报聚合/重复、节点禁言与普通互动隔离、暂停/撤销、角色范围、preview/activate/apply 和 TL4 不授予管理员。

## 当前应用入口

- 用户入口：主题、回复和用户页的举报弹窗调用 `POST /api/moderation/reports`；`/account/trust` 读取本人指标、规则、历史和制裁。
- 版主入口：`/admin/moderation` 与 `/admin/moderation/[number]` 调用案件处置、结案和制裁撤销 Route Handler，服务端按案件节点重新授权。
- 管理员受控接口：`/api/admin/governance/roles` 管理角色；`/api/admin/governance/trust/rules` 创建/查询规则，子路由执行 preview/activate；批次路由查询进度，用户路由只授予或撤销 TL4。
- 这些是同源 Web 应用内部接口，不是承诺稳定兼容的公开 OpenAPI；`v0.11.0` 的后台界面必须复用领域服务，不得绕过它们直接写表。

## 回退与演进

- 回退应用时保留全部治理、制裁和信任表；旧版本不会执行新动作，但有效暂停账号仍因 User.status 被旧授权拒绝。
- 不能通过回退删除制裁或历史。紧急恢复必须使用受审计的撤销服务。
- `v0.11.0` 可以在不改变本 ADR 领域合同的前提下增加完整用户搜索、案件筛选、角色和规则配置界面。
- 专业声誉和交易信用继续使用独立模型，不能复用 TL 字段。
