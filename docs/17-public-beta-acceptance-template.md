# 公开 Beta 人工验收模板

本文用于 `v0.13.0` 邀请测试和生产候选验收。每次安装、升级或恢复演练复制一份记录到发布工单或私有运维系统；不要把密码、Cookie、Token、完整邮箱、连接串或备份密钥写入公开 Issue。

| 字段 | 记录 |
| --- | --- |
| 验收人 / 日期 |  |
| 应用版本 / Git SHA |  |

| 部署方式 | 宝塔 Compose / CLI Compose / 非 Docker |

## 2. 记录格式

每个步骤使用同一行格式：

| ID | 操作 | 预期 | 实际 / 证据 | 结果 | 问题 |
| --- | --- | --- | --- | --- | --- |
| 示例 | 访问 `/health/ready` | HTTP 200，数据库和 Redis 就绪 | 截图或日志位置 | 通过 / 失败 / 阻塞 | Issue 链接 |

问题级别只能使用：`阻断`（安全、数据损坏、无法安装/升级/恢复或核心旅程不可用）、`重要`（明显影响运营但有可靠绕行）、`增强`（不阻止 Beta）。

## 3. 首次安装

- [ ] 使用发布包中的 `compose.yml` 和 `.env.example`，镜像为精确版本而非 `latest`。
- [ ] PostgreSQL、Redis 不发布公网端口；Web 只绑定回环地址并由 HTTPS 反向代理。
- [ ] `nextbufctl init` 或等价面板配置生成独立强密钥，示例值全部替换。
- [ ] setup 一次性任务成功；setup 前 Web/Worker 不会绕过门禁启动。
- [ ] `/setup` 创建唯一首位管理员；重复请求被拒绝。
- [ ] 删除 `SETUP_TOKEN` 并重启 Web，`doctor` 全部通过。
- [ ] `Powered by NextBuf` 在桌面和移动端可见且链接正确。

## 4. 身份与核心社区旅程

- [ ] 邀请或开放注册、邮箱验证、登录、退出、找回密码和旧会话撤销。
- [ ] 修改昵称、`@username`、简介、隐私和头像；旧用户名正确重定向。
- [ ] 创建主题、Markdown 预览、上传图片/文本/PDF/ZIP，并验证刷新后附件仍在。
- [ ] 回复、引用、提及、编辑、软删除和恢复；楼层不重排。
- [ ] 点赞、收藏、关注、阅读状态、搜索和通知未读/已读旅程。
- [ ] 390、1024、1440 宽度无水平溢出；键盘可跳到主要内容并完成关键表单。

## 5. 治理与后台

- [ ] 普通用户能举报主题、回复和用户，不能访问管理 API。
- [ ] 节点版主只能处理所属节点；全局版主和管理员作用域正确。
- [ ] 制裁在服务端阻止对应发布行为，撤销后恢复。
- [ ] 高风险管理员操作要求当前 Session 二次验证和确认文本。
- [ ] 站点设置、Provider 诊断、审计查询/导出和 Worker 页面可用且不泄漏密钥。
- [ ] 停止 Worker 或制造队列积压后后台显示真实告警，不伪造在线状态。

## 6. 升级

- [ ] 记录旧版本、目标版本、当前迁移和镜像 Digest，确认发布说明允许直接升级。
- [ ] 升级前 `doctor` 通过，备份归档校验成功并复制到异机。
- [ ] `nextbufctl upgrade <精确版本>` 自动拉取、停写、运行目标 setup 并恢复 Web/Worker。
- [ ] 升级后版本接口、管理员、用户会话、主题、回复、附件、通知和后台仍可用。
- [ ] 数据库迁移无失败/未知项，`runtime.initialized` 为目标版本，Outbox/Queue 持续下降。

## 7. 空卷恢复

- [ ] 在隔离环境验证归档 SHA-256，使用空 PostgreSQL/Redis/附件卷执行恢复。
- [ ] 比较恢复前后的用户、主题、回复、附件记录数量与抽样内容。
- [ ] 本地附件可读取；S3 模式已单独验证 Provider 快照/版本恢复。
- [ ] Redis 不从备份恢复，Worker 能由 PostgreSQL Outbox/调度事实重建执行状态。
- [ ] 恢复后 setup、Web、Worker、`doctor` 和首次管理员完成状态正常。

## 8. 验收结论

| 项目 | 记录 |
| --- | --- |
| 阻断问题 |  |
| 重要问题及绕行 |  |
| 增强建议 |  |
| 已知限制已确认 | 是 / 否 |
| 安装可接受 | 是 / 否 |
| 升级可接受 | 是 / 否 / 不适用 |
| 恢复可接受 | 是 / 否 |
| 最终结论 | 通过 / 有条件通过 / 不通过 |

任何未关闭的阻断问题都使本次验收不通过。自动 CI、axe、容器冒烟和 `doctor` 是必要证据，但不能替代真实域名、SMTP、对象存储、备份位置和运营角色下的人工验收。
